_WinAPI_BroadcastSystemMessage

Sends a message to the specified recipients

#include <WinAPISys.au3>
_WinAPI_BroadcastSystemMessage ( $iMsg [, $wParam = 0 [, $lParam = 0 [, $iFlags = 0 [, $iRecipients = 0]]]] )

ParametersParamètre

$iMsg The message to be sent.
$wParam [optional][optional] False - i, v, x ... The message-specific information.
$lParam [optional][optional] False - i, v, x ... The message-specific information.
$iFlags [optional][optional] False - i, v, x ... The broadcast option. This parameter can be one or more of the following values:
    $BSF_ALLOWSFW
    $BSF_FLUSHDISK
    $BSF_FORCEIFHUNG
    $BSF_IGNORECURRENTTASK
    $BSF_NOHANG
    $BSF_NOTIMEOUTIFNOTHUNG
    $BSF_POSTMESSAGE
    $BSF_QUERY
    $BSF_SENDNOTIFYMESSAGE
$iRecipients [optional][optional] False - i, v, x ... The recipients of the message. This parameter can be one or more of the following values:
    $BSM_ALLCOMPONENTS (Default)
    $BSM_ALLDESKTOPS
    $BSM_APPLICATIONS
    $BSM_INSTALLABLEDRIVERS
    $BSM_NETDRIVER
    $BSM_VXDS

Return ValueRemarque

Success: >0 or 0 (see below), @extended flag receives a combination of $BSM_* values identifying which recipients actually received the message.
Failure: (-1), call _WinAPI_GetLastError() to get extended error information.

Remarks<i0>

If the $iFlags parameter is $BSF_QUERY and at least one recipient returned BROADCAST_QUERY_DENY to the corresponding message, the return value is 0.

The system only does marshalling for system messages (those in the range 0 to (WM_USER-1)).
To send other messages (those >= WM_USER) to another process, you must do custom marshalling.
