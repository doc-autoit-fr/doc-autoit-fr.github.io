_WinAPI_DeviceIoControl

Sends a control code directly to a specified device driver

#include <WinAPIFiles.au3>
_WinAPI_DeviceIoControl ( $hDevice, $iControlCode [, $pInBuffer = 0 [, $iInBufferSize = 0 [, $pOutBuffer = 0 [, $iOutBufferSize = 0]]]] )

ParametersParamètres

$hDevice Handle to the device on which the operation is to be performed.
The device is typically a volume, directory, file, or stream. To retrieve a device handle, use the _WinAPI_CreateFileEx() function.
To specify a device name, use the following format:
    _WinAPI_CreateFileEx("\\.\DeviceName", ...)
$iControlCode The control code for the operation.
This value identifies the specific operation to be performed and the type of device on which to perform it.
$pInBuffer [optional][optionnel][optionnel] A pointer to the input buffer that contains the data required to perform the operation.
$iInBufferSize [optional][optionnel][optionnel] The size of the input buffer, in bytes. Default is 0.
$pOutBuffer [optional][optionnel][optionnel] A pointer to the output buffer that is to receive the data returned by the operation.
$iOutBufferSize [optional][optionnel][optionnel] The size of the output buffer, in bytes. Default is 0.

Return ValueValeur de retour

Success:Succès : True and @extended flag contains the size of the data stored in the output buffer, in bytes.
Failure:Échec : False, call _WinAPI_GetLastError() to get extended error information.
