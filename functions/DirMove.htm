DirMove

Moves a directory and all sub-directories and files.Déplace un répertoire et tous les sous-répertoires et fichiers qu'il contient.

DirMove ( "source dir", "dest dir" [, flag = 0] )

ParametersParamètres

source dir Path of the source directory (with no trailing backslash).Chemin du répertoire source (sans le slash de fin). e.g.Ex.: "C:\Path1""C:\Chemin1"
dest dir Path of the destination dir (with no trailing backslash).Chemin du répertoire de destination (sans le slash de fin). e.g.Ex.: "C:\Path_Copy""C:\Chemin_copie"
flag [optional][optionnel] this flag determines whether to overwrite files if they already exist:ce paramètre détermine s'il faut écraser les fichiers existants.
    $FC_NOOVERWRITE (0) = (default) do not overwrite existing files    $FC_NOOVERWRITE (0) = (par défaut) n'écrase pas les fichiers existants
    $FC_OVERWRITE (1) = overwrite existing files    $FC_OVERWRITE (1) = écrase les fichiers existants

Constants are defined in FileConstants.au3.

Return ValueValeur de retour

Success:Succès : 1.Retourne 1 si la fenêtre existe.
Failure:Échec : 0 if there is an error moving the directory.Retourne 0 s'il y a une erreur de déplacement du répertoire.

RemarksRemarques

If the source and destination are on different volumes or UNC paths are used then a copy/delete operation will be performed rather than a move.Si la source et la destination sont sur des volumes différents ou si des chemins UNC sont utilisés alors un copier/supprimer sera effectué au lieu d'un déplacement.

If the destination already exists and the overwrite flag is specified then the source directory will be moved inside the destination.Si la destination existe déjà et que l'indicateur d'écrasement est spécifié, alors le répertoire source sera déplacé à l'intérieur de la destination.

Because AutoIt lacks a "DirRename" function, use DirMove() to rename a folder!Puisque AutoIt n'a pas la fonction "DirRename" , utilisez DirMove() pour renommer un dossier!

DirMove(@MyDocumentsDir, @TempDir & "\Backups\MyDocs")
