PixelSearch

Searches a rectangle of pixels for the pixel color provided.Recherche dans un rectangle de pixels, un pixel de couleur donnée.

PixelSearch ( left, top, right, bottom, color [, shade-variation = 0 [, step = 1 [, hwnd]]] )

ParametersParamètres

left left coordinate of rectangle.Abscisse du coin NW du rectangle.
top top coordinate of rectangle.Ordonnée du coin NW du rectangle.
right right coordinate of rectangle.Abscisse du coin SE du rectangle.
bottom bottom coordinate of rectangle.Ordonnée du coin SE du rectangle.
color Color value of pixel to find (in decimal or hex).Valeur de la couleur du pixel à trouver (en décimal ou hexadécimal).
shade-variation [optional] A number between 0 and 255 to indicate the allowed number of shades of variation of the red, green, and blue components of the color.Un nombre entre 0 et 255 pour indiquer le nombre de variation de nuances autorisées des composantes rouge, verte et bleue de la couleur. Default is 0 (exact match).Par défaut 0 (correspondance exacte).
step [optional] Instead of searching each pixel use a value larger than 1 to skip pixels (for speed).Au lieu de chercher chaque pixel, utilisez une valeur supérieure à 1 pour passer certains pixels (pour augmenter la vitesse). E.g.Par exemple : A value of 2 will only check every other pixel.Une valeur de 2 va seulement vérifier un pixel sur deux. Default is 1.Par défaut c'est 1. It is not recommended to use a step value greater than 1.
hwnd [optional] Window handle to be used.Handle de fenêtre à utiliser. Default is the desktop window.Par défaut c'est la fenêtre du bureau.

Return ValueValeur de retour

Success: a two-element array of pixel's coordinates.Retourne un tableau à deux éléments, des coordonnées du pixel. (Array[0] = x, Array[1] = y).
Failure: sets the @error flag to 1 if the color is not found.Retourne @error = 1 si la couleur n'est pas trouvée.

RemarksRemarques

The search direction varies as follows:La direction de la recherche varie selon ce qui suit :
    Left-to-Right - left < right
    Right-to-Left - right < left
    Top-to-Bottom - top < bottom
    Bottom-to-Top - bottom < top
Changing the search direction can be a useful optimization if the color being searched for frequently appears in in a specific quadrat of the search area since less searching is done if the search starts in the most common quadrant.Changer la direction de recherche peut être une optimisation utile si la couleur recherchée apparaît fréquemment dans une zone spécifique de la zone de recherche.La recherche sera réduite, si elle débute dans la zone la plus courante.

Remember, a typical display at 1024 x 768 has 786432 pixels.N'oubliez pas qu'un affichage en 1024 x 768 contient 786 432 pixels. Although PixelSearch() is optimized, narrowing the search area helps speed up the result.Bien que PixelSearch() soit optimisé, la réduction de la zone de recherche permet d'accélérer le résultat.

ExampleExemple

#include <MsgBoxConstants.au3>

; Find a pure red pixel in the range 0,0-20,300Trouve un pixel rouge pur dans la zonz (0,0)-(20,300)
Local $aCoord = PixelSearch(0, 0, 20, 300, 0xFF0000)
If Not @error Then
    MsgBox($MB_SYSTEMMODAL, "", "X and Y are: " & $aCoord[0] & "," & $aCoord[1])
EndIf

; Find a pure red pixel or a red pixel within 10 shades variations of pure redTrouve un pixel rouge pur ou un pixel rouge dans 10 variations de coloris de rouge pur
$aCoord = PixelSearch(0, 0, 20, 300, 0xFF0000, 10)
If Not @error Then
    MsgBox($MB_SYSTEMMODAL, "", "X and Y are: " & $aCoord[0] & "," & $aCoord[1])
EndIf
