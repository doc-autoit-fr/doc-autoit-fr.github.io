Référence Fonction

FileWrite

Write text/data to the end of a previously opened file.Ajoute un texte/des données à la fin d'un fichier précédemment ouvert.

FileWrite ( "filehandle/filename", "text/data" )

ParametersParamètres

filehandle/filename The handle of a file, as returned by a previous call to FileOpen().Le handle d'un fichier, renvoyé par l'utilisation de la commande FileOpen(). Alternatively, you may use a string filename as the first parameter.Alternativement, vous pouvez utiliser une chaine de caractères contenant le chemin d'accès du fichier comme premier paramètre.
text/data The text/data to write to the file.Le texte/les données à écrire dans le fichier. The text is written as is - no @CR or @LF characters are added.Le texte est écrit tel quel - aucuns @CR ou @LF ne sont ajoutés. See remark for data type.Consultez Remarques pour le type de données.

Return ValueValeur de retour

Success: 1.Retourne 1 si la fenêtre existe.
Failure: 0 if file not opened in writemode, file is read only, or file cannot otherwise be written to.Retourne 0 si le fichier n'est pas ouvert en writemode, si le fichier est en lecture seule, ou si le fichier ne peut être autrement écrit.

RemarksRemarques

The file must be opened in write mode or the FileWrite() command will fail.Le fichier doit être ouvert en écriture, sinon la commande FileWrite() échouera.

If a filename is given rather than a file handle, the file will be opened and closed during the function call.Si un chemin d'accès est donné plutôt qu'un handle, le fichier sera ouvert et fermé durant l'appel de fonction. For parsing large text files this will be much slower than using filehandles.Pour l'analyse de grands fichiers texte ce sera beaucoup plus lent que d'utiliser un handle. However, filename will be created if it does not already exist.Cependant, le fichier sera créé s'il n'existe pas déjà.

Note:Note : Do not mix filehandles and filenames, i.e., don't FileOpen() a file and then use a filename in this function.Ne pas mélanger handles et des chemins d'accès, c'est à dire, ne pas utiliser FileOpen() pour un fichier et ensuite utiliser un chemin d'accès dans cette fonction. Either use filehandles or filenames in your routines, not both.Soit utiliser les chemins d'accès les handles dans votre routine, mais pas les deux.

When writing text AutoIt will write using UTF8 (without BOM) by default. To write in another mode the file must be opened with FileOpen() and the relevant flags.

If the data is a binary type variant (and not text) it will be written to the file byte by byte.Si les données sont une variante de type binaire (et non du texte), il sera écrit dans le fichier octet par octet. Binary operation can also be forced by using FileOpen() with the binary flag.Une opération binaire peut aussi être forcée en utilisant FileOpen() avec le paramètre binaire.

ExampleExemple

#include <FileConstants.au3>
#include <MsgBoxConstants.au3>
#include <WinAPIFiles.au3>

Example()

Func Example()
    ; Create a constant variable in Local scope of the filepath that will be read/written to.Créezune constante de portée locale avec le chemin du fichier qui sera lu/écrit.
    Local Const $sFilePath = _WinAPI_GetTempFileName(@TempDir)

    ; Create a temporary file to write data to.Crée un fichier temporaire pour écrire des données.
    If Not FileWrite($sFilePath, "Start of the FileWrite example, line 1. " & @CRLF) Then
        MsgBox($MB_SYSTEMMODAL, "", "An error occurred whilst writing the temporary file.")
        Return False
    EndIf

    ; Open the file for writing (append to the end of a file) and store the handle to a variable.Ouvre le fichier en écriture (ajoute à la fin d'un fichier) et stocke le handle dans une variable.
    Local $hFileOpen = FileOpen($sFilePath, $FO_APPEND)
    If $hFileOpen = -1 Then
        MsgBox($MB_SYSTEMMODAL, "", "An error occurred whilst writing the temporary file.")
        Return False
    EndIf

    ; Write data to the file using the handle returned by FileOpen.Écrit des données dans le fichier à l'aide du handle retournée par FileOpen.
    FileWrite($hFileOpen, "Line 2")
    FileWrite($hFileOpen, "This is still line 2 as a new line wasn't appended to the last FileWrite call." & @CRLF)
    FileWrite($hFileOpen, "Line 3" & @CRLF)
    FileWrite($hFileOpen, "Line 4")

    ; Close the handle returned by FileOpen.Ferme le handle renvoyé par FileOpen.
    FileClose($hFileOpen)

    ; Display the contents of the file passing the filepath to FileRead instead of a handle returned by FileOpen.Affiche le contenu du fichier en passant le chemin du fichier à FILEREAD au lieu du handle retournée par FileOpen.
    MsgBox($MB_SYSTEMMODAL, "", "Contents of the file:" & @CRLF & FileRead($sFilePath))

    ; Delete the temporary file.Supprimez le fichier temporaire.
    FileDelete($sFilePath)
EndFunc   ;==>Example
