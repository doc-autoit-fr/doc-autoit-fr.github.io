GUICtrlSendMsg

Send a message to a control.Envoie un message à un contrôle.

GUICtrlSendMsg ( controlID, msg , wParam, lParam )

ParametersParamètres

controlID The control identifier (controlID) as returned by a GUICtrlCreate...() function, or -1 for the last created control.L'identifiant du contrôle (controlID) tel que renvoyé par une fonction GUICtrlCreate...(), ou -1 pour désigner le dernier contrôle créé.
msg type of message to be send to the control as defined in the Windows control documentation.type de message à envoyer au contrôle tel que défini dans la documentation de configuration de Windows.
wParam The first param to send to the control.Le premier paramètre à envoyer au contrôle.
lParam The second param to send to the control.Le second param à envoyer au contrôle.

Return ValueValeur de retour

Success:Succès : the value returned by the SendMessage Windows API.Retourne la valeur renvoyée par l'API SendMessage Windows.
Failure:Échec : 0.Retourne 0.

RemarksRemarques

This function allows the sending of special Windows messages directly to the control using the SendMessage API.Cette fonction permet d'envoyer des messages spéciaux de Windows directement au contrôle en utilisant l'API SendMessage. It is used to enable special control features not available with the simple GUICtrlRead() and GUICtrlUpdate.... range of functions.Elle est utilisée pour activer les fonctions de contrôlse spéciaux qui ne sont pas disponibles avec le simple éventail de fonctions GUICtrlRead() et GUICtrlUpdate....

The parameters (wParam and lParam) can be an integer or a string.Les paramètres (wParam et lParam) peut être un entier ou une chaîne.

GUICtrlSendMsg() should be used for messages that have no special return types.GUICtrlSendMsg() devrait être utilisée pour des messages qui n'ont pas de types de retour spéciaux. For more advanced messages where you must be able to receive extra data you must use GUICtrlRecvMsg().Pour plus de messages avancées où vous devez être en mesure de recevoir des données supplémentaires ...vous devez utiliser GUICtrlRecvMsg().

ExampleExemple

#include <GUIConstantsEx.au3>
#include <ProgressConstants.au3>

Example()

Func Example()
    GUICreate("Marquee Progress Bar", 290, 90, -1, -1) ; An example of starting/stopping a scrolling marquee of a progress bar.Un exemple de démarrage/arrêt d'une barre de progression.
    Local $idProgress = GUICtrlCreateProgress(10, 10, 270, 20, $PBS_MARQUEE)
    Local $idStart = GUICtrlCreateButton("&Start", 10, 60, 70, 25)
    Local $idStop = GUICtrlCreateButton("S&top", 85, 60, 70, 25)

    GUISetState(@SW_SHOW)

    ; Loop until the user exits.Boucle jusqu'à ce que l'utilisateur quitte.
    While 1
        Switch GUIGetMsg()
            Case $GUI_EVENT_CLOSE
                ExitLoop

            Case $idStart
                GUICtrlSendMsg($idProgress, $PBM_SETMARQUEE, 1, 50) ; Send the message $PBM_SETMARQUEE and wParam of 1 to start the scrolling marquee.Envoie le message $PBM_SETMARQUEE et WPARAM à 1 pour démarrer le défilement.

            Case $idStop
                GUICtrlSendMsg($idProgress, $PBM_SETMARQUEE, 0, 50) ; Send the message $PBM_SETMARQUEE and wParam of 0 to stop the scrolling marquee.Envoie le message $PBM_SETMARQUEE et WPARAM à 0 pour arrêter le défilement.

        EndSwitch
    WEnd
EndFunc   ;==>Example
