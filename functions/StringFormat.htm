StringFormat

Returns a formatted string (similar to the C sprintf() function).Retourne une chaîne formatée (similaire à sprintf() en C).

StringFormat ( "format control", var1 [, ... var32] )

ParametersParamètre

format control The format and flags to use (see Remarks).Le format et les drapeaux à utiliser (Voir Remarques).
var1...var32 Up to 32 variables that will be output according to the "format control".Jusqu'à 32 variables qui seront affichées selon le paramètre "format control".

Return ValueRemarque

Returns the formatted string according to "format control" parameter.Retourne la chaîne formatée selon le paramètre "format control".

Remarks<i0>

To prevent buffer overflow, each "variable" is limited to 65535 characters.Pour éviter le débordement de la mémoire tampon, chaque "variable" est limitée à 65535 caractères.
Escape characters can be contain in the "format control" such as \n (@LF), \r (@CR), \t (@TAB].Les caractères d'échappement peuvent être utilisés dans la chaîne "format control" comme \n (@LF), \r (@CR ), \t (@TAB). So if you want to have a "\" you need to use \\, samething for "%" %%.Donc, si vous voulez avoir un "\" vous devez utiliser \\, même chose pour "%".

"variable format" is; %[flags] [width] [.precision] typeLa "variable format" est %[flags] [width] [.precision] type

If a format specification is invalid, the behaviour is undefined.Si une spécification de format est invalide, le comportement est indéfini. If you specify invalid input you can in special circumstances create unhandled exception and cause program termination.Si vous spécifiez une entrée non valide, vous pouvez créer des circonstances spéciales non gérée et provoquer la fin du programme.

    Width Specification

The second optional field of the format specification is the width specification.Le deuxième champ optionnel de la spécification du format est la largeur. The width argument is a nonnegative decimal integer controlling the minimum number of characters printed.L'argument 'width' est un entier décimal non négatif contrôlant le nombre minimum de caractères affichés. If the number of characters in the output value is less than the specified width, blanks are added to the left or the right of the values — depending on whether the – flag (for left alignment) is specified — until the minimum width is reached.Si le nombre de caractères à afficher est inférieur à la largeur spécifiée, des espaces sont ajoutés à gauche ou à droite selon que le drapeau (pour un alignement à gauche) est spécifiée, jusqu'à ce que la largeur minimale soit atteinte. If width is prefixed with 0, zeros are added until the minimum width is reached (not useful for left-aligned numbers).Si la largeur est précédé de 0, des zéros sont ajoutés jusqu'à ce que la largeur minimale soit atteinte (inutile pour les nombres alignés à gauche).

The width specification never causes a value to be truncated.La spécification de largeur ne provoque jamais une valeur à tronquer. If the number of characters in the output value is greater than the specified width, or if width is not given, all characters of the value are printed (subject to the precision specification).Si le nombre de caractères de la valeur de sortie est supérieure à la largeur spécifiée, ou si la largeur n'est pas donnée, tous les caractères de la valeur sont affichés (sous réserve de la spécification de précision).

    Precision Specification

The third optional field of the format specification is the precision specification.Le troisième champ optionnel de la spécification du format est la spécification de la précision. It specifies a nonnegative decimal integer, preceded by a period (.), which specifies the number of characters to be printed, the number of decimal places, or the number of significant digits (see Table below).Il spécifie un entier décimal non négatif, précédé par un point (.), qui indique le nombre de caractères à afficher, le nombre de décimales, ou le nombre de chiffres significatifs (voir le tableau ci-dessous). Unlike the width specification, the precision specification can cause either truncation of the output value or rounding of a floating-point value.Contrairement à la spécification de largeur, la spécification de précision peut entraîner soit la troncature de la valeur de sortie, soit l'arrondi d'une valeur à virgule flottante. If precision is specified as 0 and the value to be converted is 0, the result is no characters output, as shown below:Si la précision est définie à 0 et la valeur à convertir est égale à 0, le résultat n'a aucun caractère en sortie, comme indiqué ci-dessous :

StringFormat( "%.0d", 0 ); /* No characters return */StringFormat("%.0d ", 0); /* Pas de caractères en retour */

Example()

Func Example()
    Local $iInt_Unsigned = 43951789
    Local $iInt_Negative = -43951789

    ConsoleWrite(@CRLF & "Numeric Formats" & @CRLF)

    PrintFormat($iInt_Unsigned, "%d", "standard positive integer with no sign", 1) ; 43951789
    PrintFormat($iInt_Negative, "%d", "standard negative integer with sign", 1) ; -43951789
    PrintFormat($iInt_Unsigned, "%i", "standard integer", 1) ; 43951789
    PrintFormat($iInt_Unsigned, "%09i", "9 digits with leading zero", 1) ; 043951789
    PrintFormat($iInt_Unsigned, "%e", "scientific notation") ; 4.395179e+0074.395179e + 007
    PrintFormat($iInt_Unsigned, "%u", "unsigned integer with positive integer", 1) ; 43951789
    PrintFormat($iInt_Negative, "%u", "unsigned integer with negative integer", 1) ; 4251015507
    PrintFormat($iInt_Unsigned, "%f", "floating point") ; 43951789.000000
    PrintFormat($iInt_Unsigned, "%.2f", "floating point with 2 digits after decimal point", 1) ; 43951789.0043.951.789,00
    PrintFormat($iInt_Unsigned, "%o", "octal", 1) ; 247523255
    PrintFormat($iInt_Unsigned, "%s", "string", 1) ; 43951789
    PrintFormat($iInt_Unsigned, "%x", "hexadecimal (lower-case)", 1) ; 29ea6ad
    PrintFormat($iInt_Unsigned, "%X", "hexadecimal (upper-case)", 1) ; 29EA6AD
    PrintFormat($iInt_Unsigned, "%+d", "sign specifier on a positive integer", 1) ; +43951789
    PrintFormat($iInt_Negative, "%+d", "sign specifier on a negative integer", 1) ; -43951789

    Local $sString = "string"
    Local $sString_Long = "longstring"

    ConsoleWrite(@CRLF & "String Formats - [ ] used to show beginning/end of string" & @CRLF)

    PrintFormat($sString, "[%s]", "standard string", 1) ; [string][ chaîne ]
    PrintFormat($sString, "[%10s]", "10 chars right justified with added spaces") ; [ string][ Chaîne ]
    PrintFormat($sString, "[%-10s]", "10 chars left justified with added spaces") ; [string ][ chaîne ]
    PrintFormat($sString_Long, "[%10.8s]", "right justified but precision 8 so truncated") ; [ longer s][ Plus de ]
    PrintFormat($sString_Long, "[%-10.8s]", "left justifed but precision 8 so truncated") ; [longer s ][ plus de ]
    PrintFormat($sString, "[%010s]", "10 chars with leading zero") ; [0000string][ 0000string ]

    ConsoleWrite(@CRLF & "Date Format - each % uses a new parameter" & @CRLF)

    ConsoleWrite('"%02i\%02i\%04i" 0n (1, 9, 2013) => ' & StringFormat("%02i\%02i\%04i", 1, 9, 2013) & @CRLF & @CRLF)
EndFunc   ;==>Example

Func PrintFormat($vVar, $sFormat, $sExplan, $iTab = 0)
    ConsoleWrite('"' & $sFormat & '" on ' & $vVar & @TAB & ' => ' & StringFormat($sFormat, $vVar))
    If $iTab Then ConsoleWrite(@TAB)
    ConsoleWrite(@TAB & " ; " & $sExplan & @CRLF)
EndFunc   ;==>PrintFormat
