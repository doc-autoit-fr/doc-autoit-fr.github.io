ControlSend

Sends a string of characters to a control.Envoie une chaîne de caractère à un contrôle.

ControlSend ( "title", "text", controlID, "string" [, flag = 0] )

ParametersParamètres

title The title/hWnd/class of the window to access.Le titre/handle/class de la fenêtre à atteindre. See Title special definition.Consultez Définiton spéciale de titre.
text The text of the window to access.Le texte de la fenêtre à accéder. See Text special definition.Consultez Définition spéciale de Texte .
controlID The control to interact with.Le contrôle avec lequel interagir. See Controls.Consultez Controls.
string String of characters to send to the control.Chaîne de caractères à envoyer au contrôle.
flag [optional][optionnel] Changes how "keys" is processed:Change la manière dont les touches sont envoyées:
    $SEND_DEFAULT (0) = Text contains special characters like + and ! to indicate SHIFT and ALT key-presses (default).qui sont utilisés pour indiquer les touches SHIFT et ALT (défaut).
    $SEND_RAW (1) = keys are sent raw.

Constants are defined in "AutoItConstants.au3".

Return ValueValeur de retour

Success:Succès : 1.Retourne 1 si la fenêtre existe.
Failure:Échec : 0 if window/control is not found.Retourne 0 si la fenêtre/le contrôle n'est pas trouvé.

RemarksRemarques

ControlSend() works in a similar way to Send() but it can send key strokes directly to a window/control, rather than just to the active window.ControlSend() fonctionne de manière similaire à Send, mais il peut envoyer des appuis de touche directement à une fenêtre ou un contrôle, au lieu de la fenêtre active seulement.

ControlSend() is only unreliable for command prompts as that works differently to normal windows (seems to check physical states rather than accepting the keystroke messages).ControlSend() est instable pour les fenêtres de commande (DOS) car elles fonctionnent différemment d'une fenêtre normale (Semble vérifier l'état physique plutôt qu'accepter les messages de frappe clavier). For normal windows ControlSend() should be way more reliable than a normal Send() - and yes it does send shift, ctrl, alt etc.Pour une fenêtre normale, ControlSend() devrait être plus fiable, qu'un Send() normal.Et oui, il envoie Shift, Ctrl, Alt etc.

As mention in the Send() help the keyboard that send different chars when in CAPS LOCK and using the Shift Key cannot be simulated. An example is the Czech Keyboard. A good workaround is to use the ControlSetText().Une bonne manière de contourner le problème, est d'utiliser ControlSetText().

The control might first need to be given focus with the ControlFocus() command, specially when referencing an controlID created by the script itself.Le contrôle peut en premier avoir besoin du focus avec la commande ControlFocus(), spécialement lorsqu'il se réfère à un controlID créer par le script lui même.

Opt("SendKeyDelay",...) alters the length of the brief pause in between sent keystrokes.
Opt("SendKeyDownDelay",...) alters the length of time a key is held down before being released during a keystroke.

ExampleExemple

Example()

Func Example()
    ; Run NotepadExécute le Bloc-notes
    Run("notepad.exe")

    ; Wait 10 seconds for the Notepad window to appear.Attend 10 secondes pour que la fenêtre du bloc-notes fenêtre apparaisse.
    Local $hWnd = WinWait("[CLASS:Notepad]", "", 10)

    ; Wait for 2 seconds.Attend 2 secondes.
    Sleep(2000)

    ; Send a string of text to the edit control of Notepad.Envoie une chaîne de texte au contrôle d'édition du Bloc-notes. The handle returned by WinWait is used for the "title" parameter of ControlSend.Le handle renvoyé par WinWait est utilisé pour le \"titre\" paramètre de ControlSend.
    ControlSend($hWnd, "", "Edit1", "This is some text")

    ; Wait for 2 seconds.Attend 2 secondes.
    Sleep(2000)

    ; Close the Notepad window using the handle returned by WinWait.Ferme la fenêtre du Bloc-notes en utilisant le handle renvoyé par WinWait.
    WinClose($hWnd)

    ; Now a screen will pop up and ask to save the changes, the classname of the window is calledMaintenant, une fenêtre s'ouvre et demande d'enregistrer les modifications, le nom de classe de la fenêtre est appelé
    ; "#32770" and simulating the "TAB" key to move to the second button in which the "ENTER" is simulated to not "save the file""# 32770" et la simulation de la touche "TAB" pour passer au deuxième bouton dans lequel le "ENTER" est simulé pour ne pas enregistrer le fichier
    WinWaitActive("[CLASS:#32770]")
    Sleep(500)
    Send("{TAB}{ENTER}")
EndFunc   ;==>Example
